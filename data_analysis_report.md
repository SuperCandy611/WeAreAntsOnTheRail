## 欄位結構

### 舊格式（2005/01/01 ~ 2019/04/22）

| 欄位名稱 | 說明 |
|---------|------|
| `BOARD_DATE` | 乘車日（YYYYMMDD，整數） |
| `TKT_BEG` | 舊式站碼流水號（2, 4, 6 ... 292），**與新格式不同** |
| `STOP_NAME` | 車站名稱（中文） |
| `進站` | 進站人數 |
| `出站` | 出站人數 |

### 新格式（2019/04/23 ~ 2025/12/31，CSV；2026，JSON）

| 欄位名稱 | 說明 |
|---------|------|
| `trnOpDate` | 乘車日（YYYYMMDD，整數） |
| `staCode` | 車站代碼（整數，如 900 = 基隆） |
| `gateInComingCnt` | 進站人數 |
| `gateOutGoingCnt` | 出站人數 |

## 最大技術挑戰：兩種格式的站碼不相容

- 舊格式的 `TKT_BEG` 為內部流水號（2、4、6 … 292）
- 新格式的 `staCode` 為 4 碼站代碼（900、910 … ），與車站 JSON 直接對應
- **解決方式**：舊格式有 `STOP_NAME`，以站名為橋梁建立 `TKT_BEG → staCode` 對照表，可全程式化處理

## 核心指標：淨流量

$$\text{淨流量} = \text{進站人數} - \text{出站人數}$$

| 淨流量 | 詮釋 |
|--------|------|
| 負值（出 > 進） | 此站為「出發地」：人往外走，城市掏空 |
| 正值（進 > 出） | 此站為「目的地」：人湧入，可能是家鄉 |

三節連假前夕，大城市（台北）應出現大規模負值；
假期尾聲，應翻轉為正值（大家回台北）。

## 規劃功能

1. **節假日流動地圖（主打）**
   - 選擇假期類型與年份
   - 地圖上每站以顏色（紅=淨流出、藍=淨流入）＋泡泡大小呈現
   - 時間軸滑桿：看流向如何逐日翻轉

2. **重大事件觀察**
   - COVID-19（2020–2022）對各站的衝擊（斷崖式下跌）
   - 比較 COVID 前後同假期的差異

3. **車站 20 年趨勢**
   - 選一個站，看 2005–2026 年均流量變化
   - 可疊加假期與 COVID 時間標記

4. **區域流量對比**
   - 依 supp_info 區域標籤聚合
   - 大台北 / 高雄 / 花東 等在三節的流向比較
